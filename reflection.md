# Reflection

I wrote descriptive comments above each function and then shaped the implementation around the projects instructions. The functions that most naturally fit that workflow were `clean_column_names()` and `handle_missing_values()`, because both start from a clear intent: make the headers consistent and deal with blanks or invalid numeric entries in a repeatable way. The prompts were essentially the comments themselves, such as standardizing headers with lowercase and underscores or converting invalid numeric values to missing values before filling them.

I still needed to modify the generated structure to make it fit the assignment and the dataset. For example, I chose to work with lists of row dictionaries so the script would run in a plain Python environment without extra package installation, and I added helper functions that convert invalid numeric text into missing values instead of crashing the script. I also rounded and converted `quantity` back to an integer after filling missing values, because quantities should be whole numbers in a sales file. Those changes made the pipeline cleaner and easier to explain.

This project reinforced that data cleaning is mostly about consistency. Small issues like extra whitespace, mixed capitalization, blank cells, and invalid negative values can make a dataset frustrating to use unless you define a clear rule for each one. One specific example was the `price` column: leaving non-numeric values untouched would break later analysis, so converting them to a missing value and filling them with the median created a practical and consistent result.

I also learned that AI suggestions are most useful when the instructions are specific. AI can quickly produce a starting structure for a function, but it does not automatically know the best rule for a particular dataset or assignment. The human part is deciding whether to drop rows, fill values, rename variables, or simplify logic so the code remains readable. In this project, the AI-style draft was helpful for speed, but the final quality depended on reviewing every line and adjusting it to match the project goals.
